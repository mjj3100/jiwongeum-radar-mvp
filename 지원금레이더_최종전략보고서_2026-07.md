## 0. 총평 — 세 문장 진단

1. **제품은 이미 '팔 수 있는 상태'의 90%다.** 실API 318건 자동 수집, 진단 정확도 검증 통과(전자책 케이스 73점 vs 프로덕션 70점), 결제·이용권·관리자 콘솔까지 갖췄다. 남은 것은 기능이 아니라 가격 확정과 포장이다.
2. **진짜 병목은 코드가 아니라 유입과 신뢰다.** 지금 랜딩에 도달하는 사람이 0명에 가깝고, 도달해도 "이 점수를 왜 믿지?"에 답할 사회적 증명이 0건이다. 완성도 작업은 이 두 문제를 해결해주지 않는다.
3. **시즌이 데드라인을 정해준다.** 8~9월 소시즌까지 약 6주, 12~3월 메인 시즌까지 약 5개월. "완성도 높이기"는 무한히 늘어나는 작업이므로, 시즌 역산 데드라인으로 강제 마감해야 한다.

**가장 경계할 함정:** 1인 개발자에게 코드 개선은 가장 편한 도피처다. Phase 2까지 끝나면 그 뒤의 시간 배분은 코드 50% / 콘텐츠·판매 50%를 강제할 것. 이 비율을 지키는지가 이 사업의 성패를 가른다.

## 1. 확정 상품 구조 (2026-07-09 세션 합의안)

### 1-1. 3티어 단건 + 계단식 업그레이드

| | **9,900원 · 레이더 체크** | **19,900원 · 공고 정합 진단** | **29,900원 · 제출 리허설** |
|---|---|---|---|
| 고객의 순간 | "나한테 맞는 게 있긴 한가?" (아이디어 단계) | "이 공고, 내가 되나? 내 계획서가 맞나?" (공고 확정 후) | "심사받기 전에 심사를 미리 겪어본다" (제출 직전) |
| 입력 | 11개 프로필 + 아이템 한 줄 | 공고 1건 지정 + 사업계획서 (텍스트/PDF/docx — HWP는 붙여넣기 안내) | 19,900 결과 + 수정본 |
| 산출물 | 맞춤 공고 3~5건(추천·비추천 사유) / 아이디어 예비진단(4축+축별 사유) / **전자책 포함** | 4축 본진단(점수·체크리스트 모드 분기) / 원문 근거 자격 체크 / 준비서류 체크리스트 / 위험문장 **탐지** / **보완 우선순위** | 왼쪽 전부 + 위험문장 **전수 교정문(before→after)** / **예상 심사 질문 7~10개+답변 포인트** / **마감일까지 재진단 3회(축별 점수 변화 ± 비교)** / **정식 PDF 리포트** |
| 역할 | 진입·신뢰 형성 (사실상 리스크 없는 체험) | 핵심 매출 상품 | 최고 객단가 + 불안 해소 + 발표평가 커버 |

핵심 설계 원칙:
- **티어 = 고객의 준비 단계.** 업셀이 "비싼 걸 파는 것"이 아니라 "다음 단계로 가는 것"으로 읽히게 한다.
- **보완 우선순위는 19,900에 포함** (진단의 완결성 — "뭘 고칠지"까지가 진단이다). 29,900의 경계는 "탐지 vs 교정", "점검 vs 리허설".
- 29,900 결제 페이지 카피: **"심사위원을 만나기 전에, 심사위원의 질문을 먼저 받아보세요."**
- ERRC 제거 원칙 유지: 합격 보장·선정확률 표현 절대 금지. 모든 카피는 "빠진 구멍을 제출 전에 잡는다" 프레임.

### 1-2. 업그레이드 가격 — 전액 공제

- 9,900 → 19,900: **10,000원** (하위 티어 전액 공제)
- 19,900 → 29,900: **10,000원**
- 메시지: "레이더 체크 비용 9,900원은 본진단 결제 시 전액 공제됩니다."
- 리틀리 상품 총 5종: 정가 3 + 업그레이드 2. `claim_order`에서 업그레이드 클레임 시 하위 티어 이용권 보유 검증. 9,900→29,900 직행(20,000원) 상품은 문의 발생 시에만 추가.

### 1-3. 운영 규칙 (결제 전 고지 필수)

- 진단 결과 발급 후 환불 불가(디지털 콘텐츠). 발급 전 환불은 `/admin/orders` 삭제로 처리.
- 재진단 3회는 동일 공고 한정, 해당 공고 마감일까지.
- Starter 구독 코드는 유지하되 재노출하지 않는다(`SHOW_STARTER_ON_LANDING = false` 고정). 구독 재검토는 리틀리 정기결제 지원 이후.

## 3. 재무 모델 재계산 — 기존 SOM은 폐기 대상 ⚠️

**기존 P4 모델(월 1,015만원 = 420명 × 24,200원)은 구독 300명이 전제였다. 구독을 접은 지금 이 숫자는 더 이상 유효하지 않다.** 정직하게 다시 계산하면:

- 단건 객단가 가중평균 추정: 9,900×40% + 19,900×35% + 29,900×25% ≈ **18,400원** (믹스는 가설 — 실판매로 교정)
- 생계 포함 BEP(시나리오 B, 월 450~500만원) 도달에 **월 약 250~270건** 필요
- 시즌 편중(2~3월·8~9월 검색 폭증)을 감안하면 시즌 월 400건+ / 비시즌 월 100건 안팎의 분포여야 연평균이 맞는다

**솔직한 판단:** 1인 운영 + 무료 광고 채널만으로 12개월 내 월 250건은 빠듯하다. 따라서 목표를 2단계로 나눈다.
- **생존선(1차 목표): 12~3월 시즌 중 월 100건대 초반 (월 매출 약 200만원).** 이게 확인되면 사업은 산다.
- **목표선(2차): 시즌 월 250건+.** 이건 채널이 하나 더 뚫려야(유튜브 쇼츠 or 크몽 리스팅 or 유료광고 소액) 가능한 숫자로 본다.
- LLM 토큰 실비용 측정(진단 1건당)을 Phase 3에서 반드시 기록 — 재진단 3회 마진 검증용.

## 5. GTM — 유입·신뢰 콜드스타트 해법

**원칙: 첫 10건은 랜딩이 파는 게 아니라 내가 판다.**

1. **웜 오디언스부터.** 기존 오픈카톡 상담 이력(11개 입력 항목의 원출처)이 곧 리드 리스트다. 상담했던 사람들에게 직접 안내 — "그때 상담드렸던 내용을 서비스로 만들었습니다." 첫 10~20건은 여기서 나와야 하고, 이 과정의 대화 자체가 최고의 VOC다.
2. **공개 진단 사례 = 콘텐츠 = 신뢰 증거.** 본인 케이스와 (동의받은) 초기 고객 케이스를 "OO 공고, 제출 전 진단해보니 이런 구멍이 있었다" 형식의 블로그 글로. 마케팅과 사회적 증명을 한 번에 해결한다. 네이버 블로그 주 2회, 시즌 검색어('예비창업패키지 사업계획서', '소상공인 지원금 자격') 정조준.
3. **크몽 이중 활용.** 전자책과 진단 리포트를 크몽에도 리스팅 — 크몽 트래픽은 이미 지불의향이 검증된 트래픽이고, P0에서 프록시 수요 근거로 쓴 바로 그 시장이다. 수수료를 내더라도 초기엔 유입 > 마진.
4. **디스콰이엇 빌드 인 퍼블릭.** Granty-AI가 등장한 바로 그 무대에서 "진단 이후"의 차별점을 보여주는 포지셔닝 글.
5. **신뢰 메시징 상시 노출:** "브로커 아님 · 성공보수 없음 · 직접 신청 원칙 · 원문 근거 인용" — 정책자금 불신의 반사이익.

## 6. 지표 — 북극성 교체

기획안의 북극성('월간 준비방 수')은 준비방이 백로그인 현 스코프와 안 맞는다. 교체:

- **북극성: 주간 본진단(19,900+) 건수**
- **판정 지표: 9,900 → 19,900 업그레이드 전환율.** 이 구조 전체의 성립 여부를 가르는 단일 숫자. 목표 15%+, 8% 미만이면 상위 기능 추가가 아니라 업셀 카피·공제 메시지부터 수정.
- 보조: 랜딩→9,900 전환율 / 19,900→29,900 전환율 / 재진단 사용률(29,900 가치 검증) / 진단 1건 LLM 원가.
